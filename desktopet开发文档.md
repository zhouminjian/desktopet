# 桌面宠物项目开发文档

## 1. 项目概述
本项目旨在开发一个基于 Python 的 AI 桌面伙伴“Pii”。它不仅是一个漂浮在桌面上的图标，而是一个具有**真实桌面交互**、**性格成长系统**和**操作感知能力**的“数字灵魂”。

## 2. 开发环境准备
新手在开始前需确保安装以下环境：
* **语言**：Python 3.10 或更高版本。
* **UI 框架**：`PySide6`（用于实现高性能的透明窗体）。
* **监听库**：`pynput`（用于监听用户的键盘/鼠标操作频率）。

## 3. 项目结构 (Directory Structure)
```text

│   │   ├── oneeyes/    # 独眼 (点击动画)

├── core/               # 逻辑核心
│   ├── engine.py       # 宠物属性与状态管理逻辑 (PetEngine, PetState, PetStats)
│   ├── perception.py   # 键盘鼠标频率监听逻辑 (ActivityMonitor)
│   └── database.py     # 存档读写逻辑 (JSON)
├── ui/                 # 界面层
│   ├── main_window.py  # 透明窗体、动画系统、交互逻辑

├── requirements.txt    # 依赖包列表
└── main.py             # 程序入口 (PersonalitySelector, 信号连接)
```

## 4. 核心架构设计：MVC 模式
* **Model (模型)**：由 `engine.py` 负责。存储 Pii 的饱食度、心情值、经验值和性格标签。
* **View (视图)**：由 `main_window.py` 负责。透明窗口显示、动画帧切换、鼠标事件处理。
* **Controller (控制)**：`main.py` 连接两者，通过信号槽实现前后端通信。

## 5. 已实现功能清单

### 5.1 透明置顶窗口 
- 无边框、置顶、不在任务栏显示、不获取焦点
- 透明背景，宠物图片无白边
- 窗口固定尺寸 128×128
- 默认位置：屏幕右下角（任务栏上方）

### 5.2 动画系统 
- **序列帧动画**：支持多帧 PNG 序列，按帧率切换
- **动画状态**：
  - `idle` - 发呆（默认状态，2帧循环）
  - `maimeng` - 卖萌（鼠标悬停触发）
  - `oneeyes` - 独眼（左键点击触发）
  - `hungry` - 饥饿（饱食度≤35时自动切换）
  - `sleep` - 睡觉（1分钟无互动触发，3帧，每帧400ms，循环不暂停）
  - `eating` - 进食
- **动画优先级**：强制动画 > 悬停动画 > 引擎状态
- **循环暂停**：正常动画完成1个循环后暂停5秒再继续；sleep动画循环不暂停

### 5.3 交互系统 

| 操作 | 触发动画 | 冷却 | 说明 |
|------|---------|------|------|
| 鼠标悬浮 | maimeng | 无 | 进入立即触发，离开立即恢复idle |
| 鼠标离开 | idle | - | 立即恢复 |
| 左键点击（无移动） | oneeyes | 1轮+1秒 | 点击增加经验值 |
| 左键拖拽（>10px移动） | 吸附+oneeyes | 1轮+1秒 | 拖拽后吸附到最近边缘 |
| 右键 | 上下文菜单 | 无 | 显示状态/喂食/退出 |
| 冷却期内点击 | 提示"太快了" | - | 仅点击有冷却，其他操作不受限 |

- **拖拽距离判断**：用窗口位置变化（非event.pos差值）区分点击和拖拽
- **吸附逻辑**：计算到四边距离，移到最近边缘（margin=5px），已贴边则跳过
- **饥饿拒绝**：饱食度≤35时拒绝所有动画和互动（喂食除外），显示喂食提示

### 5.4 睡眠系统 
- **触发条件**：1分钟无任何互动（点击、悬浮、拖拽、右键）
- **唤醒**：任何互动立即唤醒，恢复idle + 显示唤醒提示
- **动画**：sleep序列帧，每帧400ms间隔，循环播放不暂停
- **防误触发**：启动时初始化互动时间为当前时间；存档恢复时SLEEP强制改为IDLE

### 5.5 自动动作系统 
- **定时器**：每15秒尝试触发（正态分布间隔，10-20秒之间）
- **动作权重**：
  - 85% - 发呆眨眼
  - 15% - 吸附到边缘
  - （walk/jump/roll/play_cute 已禁用）
- **条件**：仅在 IDLE 状态且不在拖拽时触发

### 5.6 对话气泡系统 
- **SpeechBubble** 组件，支持多方向箭头
- **箭头方向**：根据宠物贴边位置自动调整
  - 贴右边 → 气泡左上角，箭头右下（`right-down`）
  - 贴左边 → 气泡右上角，箭头左下（`left-down`）
  - 支持对角线箭头方向
- **跟随宠物**：宠物移动时气泡跟随更新位置
- **智能定位**：不超出屏幕边界
- **自动隐藏**：显示后按设定时长自动隐藏

### 5.7 性格选择系统 
- **PersonalitySelector** 对话框
- **宠物命名**：输入框，默认名"Pii"
- **性格标签**：活泼/温柔/幽默/傲娇/好奇/安静，互斥对立
- **性格影响**：对话语气、行为概率权重

### 5.8 养成与数值系统 
- **属性**：饱食度、心情值、经验值、等级
- **成长阶段**：幼年(BABY) → 少年(CHILD) → 青年(ADULT)
- **经验来源**：点击/抚摸、喂食
- **数值衰减**：饱食度随时间下降
- **饥饿阈值**：≤35 切换hungry动画和对话

### 5.9 感知系统 
- **ActivityMonitor**：监听键盘敲击频率
- **打字激励**：高强度打字时宠物主动送鼓励

### 5.10 数据持久化 
- **存档位置**：`%APPDATA%\DesktopPet\pii_save.json`
- **存档内容**：名称、属性、成长阶段、性格、状态、经验更新时间
- **自动存档**：30秒防抖间隔，避免频繁IO
- **退出存档**：程序退出时强制保存
- **状态恢复**：SLEEP状态强制恢复为IDLE

### 5.11 系统托盘 
- **QSystemTrayIcon**：最小化到托盘
- **托盘菜单**：显示/隐藏、退出

## 6. 关键技术点实现

### 6.1 透明置顶窗口
```python
# 窗口属性设置
self.setWindowFlags(
    Qt.FramelessWindowHint |      # 无边框
    Qt.WindowStaysOnTopHint |     # 置顶
    Qt.Tool                        # 不在任务栏显示
)
self.setAttribute(Qt.WA_TranslucentBackground)  # 透明背景
self.setAttribute(Qt.WA_ShowWithoutActivating)  # 不抢焦点
```

### 6.2 动画帧切换
- `_animation_timer`：约42ms一帧（~24FPS）
- 多帧动画：sleep每10帧(400ms)切换，其他每6帧(250ms)切换
- 单帧动画：每12帧(500ms)重绘，产生呼吸缩放效果
- `_get_current_pixmap()`：按优先级（强制>悬停>引擎状态）获取当前帧

### 6.3 拖拽与吸附
- `mousePressEvent`：记录窗口初始位置 `_drag_start_window_pos`
- `mouseReleaseEvent`：用窗口位移判断是否真正拖拽（>10px）
- `_action_snap_to_edge`：计算到四边距离，缓动移动到最近边缘
- `_start_move_animation`：60fps位移动画，支持缓动函数

### 6.4 信号系统
```python
class PetSignals(QObject):
    stats_updated = Signal(PetStats)    # 属性更新
    state_changed = Signal(PetState)    # 状态变化
    feed_requested = Signal()           # 喂食请求
    pet_requested = Signal()            # 抚摸请求
    hunger_changed = Signal(float)      # 饱食度变化
    exp_gained = Signal(int)            # 获得经验
```

## 8. 注意事项
1. **性能优化**：动画刷新率约24FPS，sleep动画400ms/帧，控制CPU占用。
2. **坐标异常**：多显示器环境下坐标计算可能出错，当前优先适配单屏幕。
3. **资源路径**：打包时需使用 `sys._MEIPASS` 处理绝对路径。
4. **存档防抖**：自动存档30秒间隔，避免频繁磁盘IO。
5. **3D模块已归档**：`assets/3d/` 和 `ui/viewer_3d.py` 已不再使用，保留供参考.
