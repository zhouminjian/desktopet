# 《桌面宠物》V1.0 技术需求说明文档

## 1. 整体技术架构 (Architecture)
为确保后续 UI 升级（如切换 QML 或 Electron）的灵活性，采用 **逻辑与表现分离** 的解耦架构。

* **表现层 (View Layer)**：使用 `PySide6` 实现。
    * **窗口属性**：无边框 (`Frameless`)、窗口置顶 (`AlwaysOnTop`)、背景透明 (`Translucent`)。
    * **渲染逻辑**：基于状态机切换 GIF 或序列帧动画。
* **逻辑层 (Logic/Engine Layer)**：纯 Python 类。
    * **状态机**：管理宠物当前行为（闲逛、睡眠、互动、特殊反应）。
    * **数值引擎**：处理心情值、饱食度、经验值的实时计算。
* **感知层 (Perception Layer)**：
    * 使用 `pynput` 或 `pyhook` 捕获全局外设输入频率信号。

---

## 2. 核心功能模块详细需求

### 2.1 桌面交互系统 (Desktop Interaction)
* **坐标管理**：程序需获取主屏幕分辨率，确保宠物初始位置在任务栏上方。
* **拖拽逻辑**：
    * 鼠标左键按下：切换为“被抓起”动画，窗口位置跟随鼠标坐标。
    * 鼠标释放：触发“掉落”动画，根据重力逻辑回归任务栏或最近的窗口顶部。
* **碰撞检测 (进阶需求)**：检测当前活动窗口的边缘坐标，实现宠物“在窗口边缘爬行”的效果。

### 2.2 宠物逻辑引擎 (Pet Engine)
* **状态切换 (State Machine)**：
    * **Idle (待机)**：随机播放呼吸、眨眼动画。
    * **Walk (巡逻)**：在屏幕底部水平移动。
    * **Triggered (被动交互)**：被点击、被拖拽、被用户操作触发。
* **数值衰减**：
    * 心情值、饱食度随时间推移按设定速率下降。
    * 当数值低于阈值（如 20%），状态机强制切换为“委屈”或“饥饿”状态。

### 2.3 用户行为感知系统 (User Awareness)
需通过底层钩子监听以下事件并转化为逻辑层的输入信号：
* **打字激励**：单位时间内敲击键盘次数超过 X 次，小猫进入“加油”状态。
* **挂机检测**：鼠标连续 Y 分钟未移动，小猫进入“撒娇”或“寻找关注”状态。
* **窗口操作**：检测到大面积窗口关闭时，触发宠物“受惊”或“抱怨”反馈。

## 3. 数据持久化方案 (Persistence)
* **存储格式**：使用本地 `config.json` 或 `sqlite` 数据库。
* **存档内容**：
    * 宠物当前阶段（幼年/少年/青年）及经验值。
    * 已选性格标签（用于行为加权）。
    * 最后一次离线的时间戳（用于计算再次登录时的数值补扣）。

---

## 4. 资源配置规范
开发人员需支持以下资源配置方式，以便后续无缝改版：
* **动画定义**：支持加载 `.gif` 或按索引排列的 `.png` 序列。
* **映射表**：建立 `state -> animation_file` 的映射配置文件。
* **对话库**：独立文本文件，支持根据不同性格标签（如：活泼、幽默、温柔）随机抽取气泡对白。

---

## 5. V1.0 验收标准 (MVP Criteria)
1.  **稳定性**：程序运行内存占用需低于 100MB（Python 环境下）。
2.  **交互性**：小猫能被正常拖拽并正确响应点击。
3.  **视觉**：背景必须完全透明，不产生矩形白边。
4.  **闭环**：宠物能根据用户打字频率做出至少一种主动反馈。
